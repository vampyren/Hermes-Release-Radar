# Roadmap

## Near term

- Add a separate GitHub Pages/public demo mode that can be regenerated by GitHub Actions without exposing Rex's local state.
- Add a scheduled/manual GitHub Actions workflow that regenerates public static output and commits only generated public files when content changes.
- Package the repo cleanly with stable install/update instructions.
- Add an install script only if Rex wants one; keep manual commands as the canonical transparent path.
- Add a small smoke-test script for local verification.
- Keep the help page rendered and readable on desktop and mobile.

## Later

- Optional config file for custom Hermes checkout/runtime paths.
- Optional authenticated LAN mode, only if needed.
- Better release-card grouping based on learned Rex relevance.
- Exportable update reports for Obsidian.
- More robust rendered Markdown pipeline for docs/help pages.

## Must not drift

- Do not turn Release Radar into an updater.
- Do not bind outside localhost by default.
- Do not publish Rex's local `state.json`, review markers, helper status, service API, local paths, or installed checkout details to GitHub Pages.
- Public GitHub Pages output must be a demo/static public snapshot, not Rex's private installed-vs-upstream page.
- Do not hide raw commit audit data.
- Do not show already-installed release bodies as pending updates.
